# サウンドテーブルテニス (STT) オンライン

サウンドテーブルテニス (STT) は、視覚障害者のために考案された卓球競技です。
このプロジェクトは、STTの公式ルールに準拠し、**視覚障害者（モバイル/PCのスクリーンリーダー利用者）も健常者も共に同じ条件で対戦できる**ことをコンセプトとした、GoバックエンドおよびWeb Audio API搭載のリアルタイムブラウザゲームです。

---

## 🌟 主な特徴

1. **Web Audio API による3D立体音響システム**
   * ボールの左右の移動をステレオパンニングで、奥行き（相手コート、ネット、自分コート）を音量と音質（ローパスフィルターによるこもり具合）の変化でシミュレートします。
   * 金属球が4個入ったSTT公式ボール特有の「シャラシャラ」という転がり音をオシレーターとノイズを用いてリアルタイムに合成します。
   * 木製ラケットで打球した時の「コン」、フレームに当たった時の「カツ」、ネットに当たった時の「ポス」など、素材感のある衝突音を再現しています。

2. **完全アクセシビリティ（PC・スマホ両対応、スクリーンリーダー＆音声ガイド）**
   * Web Speech API (`SpeechSynthesis`) による自動音声案内（実況）を搭載。審判のコールや、サーブ前の掛け合い（「いきます」「はい」）をすべて音声で実況します。
   * スクリーンリーダー（iOSのVoiceOver、AndroidのTalkBack）の操作を妨げないセマンティックなHTML構造とARIA属性設計。
   * パソコンではキーボードの最小限のキー（左右矢印とスペースキー）、スマートフォンでは「端末の傾き（チルト）」と「画面タップ」のみで直感的に対戦プレイまで完結します。

3. **Go ＋ WebSocket によるリアルタイムオンライン・クロスプラットフォーム対戦**
   * 1台のPCでのCPU対戦に加え、GoバックエンドによるWebSocket通信を利用したオンライン対戦に対応。
   * パソコン同士だけでなく、**「パソコン vs スマートフォン」**、**「iPhone vs Android」**といったデバイスの壁を越えたリアルタイム対戦（クロスプレイ）が可能です。
   * ゲームステート（サーブ制限時間、デュース判定、得点処理）はGo側で厳密に判定・同期されるため、ラグがあっても一貫したゲーム進行が可能です。

4. **プレミアムな視覚デザイン（健常者向け）**
   * ダークモードを基調としたネオン・サイバーパンク調の美しいグラフィック。スマートフォンの表示にも完全対応したレスポンシブ設計。
   * ボールの軌跡や、衝突時にコート上に広がる「音の波紋（パルス）」を可視化。音の動きを視覚的にも体感できます。
   * **STT公式規格・寸法ガイドの可視化**: 卓球台の正確な寸法（長さ274cm、幅152.5cm、サイドフレーム60cm、ネット高さ15.25cm、ネット下の隙間4.2cm）を、ゲーム画面内のネオン調寸法線と、右側のリアルタイム寸法規格サイドパネル（SVG図解付き）で視覚的に表現しました。見える人にとってもSTTの独特なルールスケールが一目で理解できます。

## 🎮 遊び方・操作方法

使用するデバイスやアクセシビリティ設定に合わせて、最適な操作方法を選択できます。

### 1. パソコンでの操作（標準キーボード）
* **左矢印キー (`ArrowLeft`)**: ラケットを左に移動
* **右矢印キー (`ArrowRight`)**: ラケットを右に移動
* **スペースキー (`Space`)**: アクション（サーブの準備、相手への返答、メニュー決定、ラリー中の打ち返し）
* **エスケープキー (`Escape`)**: ゲームの一時停止 / メニューに戻る

### 2. スマートフォンでの操作（チルト ＋ 画面タップ）
スマホ単体でプレイする場合、タッチドラッグ操作がスクリーンリーダーにブロックされるのを防ぐため、以下の操作になります。
* **ラケットの移動（チルト操作）**:
  * スマートフォン本体を**左右に傾ける**ことで、ラケットが左右に移動します。
  * プレイ開始時に、その時の手の持ち角度に合わせて**自動的にキャリブレーション（水平補正）**が行われます。
  * プレイ中、持ち方を変えたくなったら画面下部の**「チルトをリセット」**ボタンをタップ（またはダブルタップ）することで、いつでも中心位置を補正できます。
* **アクション（決定/サーブ/打ち返し）**:
  * **画面全体のどこでもタップ**（VoiceOverやTalkBackなどの**スクリーンリーダー起動時は画面の任意の場所をダブルタップ**）することで、スペースキーと同じアクションが発生します。
* **設定の切り替え**:
  * 開始時のウェルカム画面で「スマートフォンの傾き（チルト）で操作する」のチェックを外すと、スマホに接続した外付けのBluetooth物理キーボード（パソコンと同じ操作）で遊ぶことも可能です。

### STT公式ルールに基づくプレイフロー
1. **主審の「プレー」宣告**でラリーの準備が始まります。
2. **サーバー**: スペースキーまたは画面タップ（ダブルタップ）を押してボールを静止させ、準備ができたら「いきます」と発声します。（10秒以内）
3. **レシーバー**: サーバーの「いきます」に対してキー/タップで「はい」と返答します。（5秒以内）
4. **サーバー**: 返答から5秒以内にキー/タップでサーブを打ちます。（5秒以内）
5. **ラリー開始**: ボールがネットの下（テーブル面上4.2cmの隙間）を通って転がってくるので、音（左右の定位や音量）を頼りにラケットを動かしてタイミングよく打ち返します。
6. **得点判定**: ボールが相手の守備コート内で「停止」するか、エンドフレーム（奥の壁）の内側に当たって「セーフ」になれば得点です。フレームを越えて飛び出たり（アウト）、ネットに引っかかったり、ラケット以外の場所に当たると相手の得点になります。

## 🚀 起動方法

### 方法1：実行ファイルから起動（推奨・最も簡単）
1. プロジェクトルートにある `stt.exe` をダブルクリックして実行します。
2. サーバーが起動し、自動的にデフォルトのブラウザで `http://localhost:8080` が開きます。

### 方法2：Goソースコードから起動（開発者向け）
1. ターミナルでプロジェクトディレクトリに移動します。
   ```bash
   cd stt
   ```
2. 以下のコマンドでサーバーを起動します。
   ```bash
   go run main_server.go
   ```
3. サーバーが起動すると、自動的にブラウザで `http://localhost:8080` が開きます。

## 🌐 別のパソコン・スマホと対戦する場合の手順（ローカルネットワーク対戦）

同じWi-Fi（ローカルネットワーク）に接続された別のPCやタブレット・スマートフォンと対戦（クロスプレイ）することができます。

### 1. サーバーPC（親機）の準備
1. 親機となるPCで `stt.exe` を起動します。
2. 親機PCのローカルIPアドレス（IPv4）を調べます。
   * **Windowsの場合**: コマンドプロンプトを開き、`ipconfig` と入力して実行。「IPv4 アドレス」の値を確認します（例: `192.168.1.15`）。
   * ※セキュリティ警告が出た場合は、プライベートネットワークでの通信を許可してください。

### 2. 対戦相手（子機PC または スマートフォン）の接続
1. 対戦相手のPCやスマートフォンを、親機と同じWi-Fiネットワークに接続します。
2. 子機のWebブラウザを開き、アドレスバーに **「親機PCのIPアドレス:8080」** を入力してアクセスします。
   * 例: `http://192.168.1.15:8080`
   * ※スマートフォンのチルトセンサーを使用する場合、ブラウザによっては安全な接続（HTTPS）またはローカルネットワーク通信の許可が必要です。
3. 画面が表示されたら、「オンライン対戦」を選択します。

### 3. ルームに入って対戦開始
1. 親機・子機の双方で、ルームIDを**空欄のまま**「ルームに入る」ボタンを押します。（特定の相手とだけ遊びたい場合は、両方に同じルームIDを入力してボタンを押してください。）
2. これで自動的にマッチングし、対戦が始まります。
